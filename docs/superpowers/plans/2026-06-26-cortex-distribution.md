# Cortex Distribution — npm publish — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the toolkit a one-command installable npm package (`@n1x-technologies/cortex`) with a light base install, an MIT license, and a reproducible CI-on-tag release.

**Architecture:** Reshape `toolkit/package.json` into a public scoped package; move the heavy `@xenova/transformers` to an optional peer dependency so the base install stays light; add a shebang so the `cortex` bin runs after a global install; add MIT license, two GitHub Actions workflows (test-on-PR, publish-on-tag), npm-flow docs, and guard tests that fail if the package ever becomes unpublishable.

**Tech Stack:** Node ESM + TypeScript, vitest, npm (scoped publish), GitHub Actions.

## Global Constraints

- **Package name:** `@n1x-technologies/cortex`. The installed **binary name stays `cortex`** (`bin.cortex`).
- **Version:** `0.1.0`. **License:** `MIT`, `Copyright (c) 2026 N1X Technologies`. **Engines:** `node >=18`.
- **`publishConfig.access` = `"public"`** (scoped packages are private by default). **`files` = `["dist/"]`**.
- **`@xenova/transformers`** (`^2.17.2`) lives in `peerDependencies` **and** `devDependencies`, with `peerDependenciesMeta["@xenova/transformers"].optional === true`. It is **NOT** in `dependencies`.
- **Shebang:** `#!/usr/bin/env node` is the **first line** of `toolkit/src/cli.ts`.
- **Node ESM:** relative imports use explicit `.js` extensions; tests run from `toolkit/` (`npm test`, vitest); build is `npm run build`.
- **CI:** `release.yml` triggers on tags `v*`, runs test+build, guards that the tag equals `package.json` version, then `npm publish --access public` with `NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}`. `ci.yml` runs test+build on push/PR. Both use `working-directory: toolkit`.
- **Stacked on the Phase 6 semantic-layer branch.** Do not modify semantic logic beyond the `commands/embed.ts` error-message refinement in Task 2.

```ts
import { readFileSync, existsSync } from 'node:fs';
import { fileURLToPath } from 'node:url';

const here = fileURLToPath(new URL('.', import.meta.url));   // toolkit/test/

describe('package is publishable', () => {

    expect(pkg.license).toBe('MIT');
    expect(pkg.version).toBe('0.1.0');

    expect(cli.startsWith('#!/usr/bin/env node\n')).toBe(true);
  });
  it('ships an MIT LICENSE file at the repo root', () => {
    const license = join(here, '..', '..', 'LICENSE');
    expect(existsSync(license)).toBe(true);
    expect(readFileSync(license, 'utf8')).toMatch(/MIT License/);
  });
});
```

```yaml
  test:
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: toolkit
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
      - run: npm ci
      - run: npm test
      - run: npm run build
```

- [ ] **Step 4: Create `.github/workflows/release.yml`**

```yaml
name: release
on:
  push:
    tags: ['v*']
jobs:
  publish:
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: toolkit
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          registry-url: https://registry.npmjs.org
      - run: npm ci
      - run: npm test
      - run: npm run build
      - name: Verify tag matches package version
        run: |
          TAG="${GITHUB_REF_NAME#v}"
          PKG="$(node -p "require('./package.json').version")"
          test "$TAG" = "$PKG" || { echo "tag $TAG != package version $PKG"; exit 1; }
      - run: npm publish --access public
        env:
          NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}
```
